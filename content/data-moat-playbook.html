Apple HealthKit

Central aggregator ‚Äî steps, HR, ECG, sleep, workouts, and 3rd-party data
Free SDK Apple Review
‚ñæ
API / SDK Access
HealthKit Framework (iOS/watchOS native). On-device only ‚Äî data never leaves the phone without user consent. No REST API. Must be a native iOS app or use HealthKit via React Native/Flutter bridges.

Apple Developer Program ($99/yr). HealthKit entitlement required in app. App Store Review enforces HealthKit usage guidelines. Health-related apps get extra scrutiny ‚Äî 2‚Äì4 week review process.

$99/year Apple Developer Program. No per-user data fees. HealthKit SDK is free.
Data Available
Heart rate, HRV (SDNN), ECG waveforms (Apple Watch), steps, distance, workouts, sleep analysis, respiratory rate, blood oxygen, body measurements, nutrition, menstrual tracking, plus all 3rd-party data written to HealthKit (Oura, WHOOP, etc.).

Dependencies
iOS app required. Apple Developer Program membership. App Store approval. HealthKit is iOS-only ‚Äî no Android equivalent. Acts as a meta-aggregator for all health data on the user's phone.
Strategic Note
HealthKit is the aggregation layer, not a unique data source. Its value: a single integration gets you Apple Watch + all 3rd-party device data users have synced. Critical for user reach but doesn't provide data you can't get elsewhere. Score: 45/125 (3 √ó 5 √ó 3)

Critical Path Analysis

What blocks what. What runs independently. Where the bottlenecks are.

üöß

Blocker: Dexcom Application Review (4‚Äì8 weeks)

Longest single wait in the entire plan. Application must be submitted Day 1 to hit Week 12 live target. Dexcom is selective ‚Äî the application needs to demonstrate clinical value, not just data aggregation. Mitigation: Build sandbox integration during wait. Prepare OAuth flow, data models, and CGM ‚Üí causal DAG mapping so dev starts immediately on approval. If rejected, Abbott FreeStyle Libre / ADC program is fallback (even slower).

üöß

Blocker: Garmin License Negotiation (2‚Äì4 weeks after approval)

Garmin requires a commercial license for BBI data. Two-stage gate: developer approval (2‚Äì4 weeks) then license negotiation (2‚Äì4 weeks). Budget approval for $10K‚Äì$50K/year needed. Mitigation: Start with evaluation environment (may be free). Prepare spectral HRV pipeline in advance. If license is too expensive, Polar BBI via H10 is a partial substitute (but narrower user base).

‚ö†Ô∏è

Dependent: Vim Lab Results ‚Üí Medication Lists ‚Üí Diagnoses

Sequential integration ‚Äî lab results are highest value and should ship first. Medication list parsing depends on the FHIR data model established for labs. Diagnosis codes and clinical notes come last (lower priority, higher complexity with unstructured data). These are internal dependencies ‚Äî Vim partnership is already active, no external blocker.

‚ö†Ô∏è

Dependent: Apple HealthKit ‚Üí App Store Review

HealthKit integration requires iOS native code and App Store approval. Health apps get extra scrutiny ‚Äî 2‚Äì4 weeks for review. Submit by Week 6 to be live by Week 8. HealthKit is a meta-aggregator ‚Äî once live, it unlocks all 3rd-party data synced to Apple Health (including Oura, WHOOP, etc. without separate integration).

‚úÖ

Independent: Polar SDK, Oura API, WHOOP API, Withings Public API

All four have zero application wait time ‚Äî free, open developer access. All four can begin integration on Day 1 and be live by Week 4‚Äì5. These are your quick wins that demonstrate multi-device capability while you wait for Dexcom and Garmin approvals. Assign one developer per two integrations (Polar+Oura, WHOOP+Withings).

‚úÖ

Independent: Vim EHR (already active)

Vim partnership is already in place. Lab result integration runs on a completely independent track from all wearable integrations. Different developer, different codebase, different data pipeline. This is the most important track ‚Äî the EHR backbone that makes all wearable data 10x more valuable.
